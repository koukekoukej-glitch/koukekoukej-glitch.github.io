♬ TL Agent / 详细展示 01/06
I. Prélude

TL Agent

面向 ARPG 研发团队的 AI Agent——把散落在引擎、配表、代码和协作平台中的操作，统一到一个对话界面

II. Allemande

统一入口

把团队的 AI 提效能力收束到一个网页入口

团队用 AI 提效的关键在于上下文能不能被持续积累、被所有人共享。每个人各自摸索 AI 的用法、最后只是大家在重复造轮子。所以要把团队的 AI 能力收束到一个入口、所有人共用同一套上下文工程，一起维护迭代

— i
零门槛入口

用大家最熟悉的网页聊天形态——类似 ChatGPT / Gemini 那种、打开网页就用、不用装环境、不用学命令行

— ii
共享的上下文工程

把项目的领域知识沉淀成一套上下文工程（详见下一节）、所有人共用同一份。每个研发同学直接让"已经懂这个项目的 AI"协助工作

— iii
无感远程操控

用户不需要切换任何心智——网页里和 Agent 对话、Agent 在背后改本地配置 / 写本地代码 / 提交 P4，透明代理层让 Agent 自己不知道在远程操作

— iv
专业用户出口

对习惯 Claude Code 这类专业工具的用户可以通过 API 接入自己的工作流；整套上下文工程是开源的，所有人可以拉到本地做二次改造，走流程合并回服务器主线、与团队成员共享

III. Courante

上下文工程

让 Agent 持续懂这个项目

游戏项目的隐性知识太多、每个人的工作经验又不同、一个人根本整理不完。所以上下文工程靠三个驱动力：团队内部分享、用户主动反馈、会话数据积累——让它越来越懂这个项目

— i
Skill 社区

每个用户都可以把自己擅长的领域知识和工作流按固定结构打包成 Skill 发布到社区、其他人订阅后实时获得更新

— ii
锐评反馈

每个会话界面有一个"锐评"按钮、用户对 Agent 表现不满时一键提交、可附加自由文本说明；提交后启动独立 Claude 实例做第一视角还原、定位上下文工程哪一环出了问题、产出可执行的修改方案

— iii
会话挖掘

会话不活跃时、后台启动独立 Agent 通读完整对话提取记忆。进入 pending 队列由会话发起者审核后入库；记忆分两类：
(i). 术语：下次会话用到时主 Agent 自己 grep 术语表命中即取
(ii). 规则：由一个后台 Agent 实时监控当前对话、命中相关记忆后"插队"塞回主 Agent 上下文

IV. Sarabande

项目专属工具

让 Agent 处理项目中的复杂格式文件

游戏研发的大量工作围绕引擎资产和配置表这些 Agent 无法原生读写的复杂格式文件展开，要让 AI 更深入地嵌入工作流，就需要有专用的工具让它能更好地理解和读写这些信息

— i
Field Graph

策划的工作主要围绕项目里 800+ 张配置表展开，表中有大量复杂字段，但没有标准 Schema。Field Graph 为所有配置表预建关系索引：覆盖所有字段的注释和外键引用关系，Agent 查表时返回的是字段的完整语义信息，不需要去猜含义

— ii
Table Writer

项目组的策划配置表有一系列固定的格式规则，直接用通用库写入容易破坏公式引用和格式。Table Writer 封装安全写入：只动被改的单元格、其余保持不变、优先调 Excel 自身保存机制保格式零损坏。写入前自动完成版本控制 checkout、写入后可触发数据校验和导出；Agent 试图绕过它直接写 Excel 会被 Hook 拦截并引导回来

— iii
UE HTTP API

引擎内部的配置数据和美术资产是二进制格式、外部程序无法直接安全修改。我们在引擎编辑器内部署 HTTP 服务插件、Agent 通过它读写引擎配置表行数据、创建编辑美术资产、生成动画蓝图文件、触发数据导出。所有操作在引擎内部原子完成——避免外部工具和引擎编辑器争抢文件锁

V. Menuet

AI 原生工作流

让 AI 成为工作流的编排者、不只是工作流里的一步

— i
飞书配置助手

策划的大量配置工作有复杂的前置依赖且需要多工种协同，我们把所有前置依赖项整理成格式化的飞书表格，美术填完资产、文案填完包装，直接在飞书群里 @TL Agent 说"帮我更新时装配置"：Agent 自动读多维表格需求源、对照本地 Excel 配表增量更新、跑校验、用专属 P4 CL 提交、回群汇报，直接就能进游戏验收，不再依赖配置同学协同

— ii
策划文档审核

策划在飞书文档里写完案子、评论区直接 @TL Agent，唤起自动审查流程，Agent 把问题以飞书评论形式挂在文档对应段落上 @ 作者、通过文档的自动归档到 wiki 对应领域。审核规则是一份可以自由扩展的 Skill，整个研发上下游都可以维护它。审查状态持久化在飞书多维表格、对人类透明、Agent 进程崩溃也不丢状态

VI. Gigue

外部工具集成

连接团队已有的工作平台

— i
即时通讯

接入 Slack、用户在群里 @TL Agent 即可使用、不需要打开网页界面；所有动作群成员可见、既保留即时讨论的协作感、也让任何人都能实时介入或打断

— ii
项目管理

接入 Jira、Agent 能查任务、找负责人、跟进度、看排期、帮用户在对话中直接获取项目管理信息

— iii
数据查询

接入线上数据仓库、用户用自然语言描述需求、Agent 生成 SQL 查询并返回结果——对比不同版本的首日付费收入、分析新版本玩家留存和行为路径、查询特定玩法的参与率和收益数据

— iv
服务端诊断

每个团队成员都有自己的个人测试服、用户排查问题时 Agent 直接访问服务端错误日志、读取运行时配置文件、查询游戏数据库、不需要找研发帮忙
